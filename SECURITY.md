# Security Policy

Clay CMS is pre-1.0 software under active development. We take security
seriously — auth, access control, and session handling are core to the
project — and we appreciate responsible disclosure.

## Supported Versions

Clay CMS has not yet reached a stable `1.0` release. Only the latest
published version receives security fixes; there is no back-porting to
earlier `0.x` releases. Pin an exact version and upgrade to pick up fixes.

## Reporting a Vulnerability

**Please do not open a public issue for security vulnerabilities.**

Report privately via GitHub's ["Report a vulnerability"](https://github.com/reeple-studio/clay-cms/security/advisories/new)
button (Security tab), or email **miguel@reeple.studio**.

Please include:

- A description of the issue and its impact
- Steps to reproduce (a minimal `clay.config.ts` / collection setup helps)
- Affected version(s)

We aim to acknowledge reports within 72 hours and will keep you updated as
we investigate. Once a fix ships, we're happy to credit you unless you'd
prefer to remain anonymous.
